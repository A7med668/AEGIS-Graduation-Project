Threat Model

☠️
Malicious / Trojanized Apps
APKs embedding hidden payloads, RATs, or credential harvesters delivered via sideloading or compromised stores.
📡
Data Exfiltration
Unauthorized transmission of tokens, credentials, and PII to C2 endpoints via covert channels.
🎭
Overlay / Phishing
UI overlay attacks that intercept credentials via fake login screens drawn over legitimate apps.
🔓
Rooted / Compromised Devices
Root access bypasses Android security boundaries, enabling full memory reads and certificate pinning defeat.
👁️
Permission Abuse
Legitimate permissions (camera, mic, location) exploited at runtime beyond declared scope.
🔑
Hardcoded Secrets
API keys, tokens, and credentials embedded in APK resources or compiled bytecode.

Core Components

🤖
Kotlin · On-Device
Android Agent (EDR)
Lightweight endpoint detection agent running in the work profile. Collects security telemetry, not content.
  • Device integrity via Play Integrity API
  • Root / bootloader detection
  • APK hash + permission diffing on install events
  • Behavior monitor: app launches, permission spikes
  • Local risk pre-scoring — sends deltas only
  • Certificate pinning for all upstream comms
🚪
FastAPI · mTLS
Ingestion Gateway
Authenticated, rate-limited entry point for all device telemetry and APK submissions.
  • Mutual TLS device authentication (cert-based)
  • Per-device rate limiting
  • Payload schema validation (Pydantic)
  • APK upload endpoint with chunked transfer
  • Publishes validated events to Kafka topics
🔬
Python · MobSF · jadx
Static Analysis Engine
Decompiles and inspects APK bytecode, resources, and manifest without execution.
  • APK unpack via apktool + jadx decompilation
  • Secret detection (regex + entropy analysis)
  • Dangerous API pattern matching
  • Obfuscation detection heuristics
  • 3rd-party tracker SDK fingerprinting
  • Manifest permission abuse scoring
  • Certificate / signature validation
💉
Frida · Android Emulator
Dynamic Sandbox
Executes APKs in an instrumented emulator and captures runtime behavior for fingerprinting.
  • Emulator-based isolated execution
  • Frida hook injection for API interception
  • Network endpoint capture (C2 detection)
  • File system access tracing
  • Crypto primitive usage logging
  • Behavior fingerprint generation (hash)
  • Anti-analysis evasion detection
🧠
Python · ML Pipeline
Risk Engine
Aggregates signals from all engines and produces a unified risk score per device and app.
  • Weighted multi-signal aggregation
  • Device risk + App risk + Behavior anomaly
  • Threat intel enrichment (VirusTotal, OTX)
  • Time-series anomaly detection
  • Severity classification: Low → Critical
  • Alert deduplication and suppression
🤖
LLM · Claude / GPT-4
AI Analyst Layer
LLM-powered module that translates raw analysis results into human-readable threat reports.
  • Natural-language threat explanation
  • Automated executive summary generation
  • Remediation recommendation engine
  • APK analysis narrative from static output
  • SOC triage assistance via chat interface
🖥️
React · Next.js
SOC Dashboard
Real-time analyst interface for device fleet visibility, alert management, and investigation.
  • Device fleet risk heatmap
  • App reputation database with history
  • Alert timeline with drill-down
  • Attack path visualization per incident
  • One-click MDM response actions
  • Analyst investigation workspace
⚡
MDM · Intune · SIEM
Response Engine
Automated and analyst-triggered enforcement actions across the device fleet via MDM integration.
  • Auto-quarantine on Critical severity
  • App install block policy push
  • Remote wipe (COPE devices)
  • Jira ticket auto-creation
  • SIEM event forwarding (Splunk / ELK)
  • Webhook support for custom playbooks
💻
Python CLI · Rich
Security Lab CLI
Command-line tool for SOC engineers and security researchers to run on-demand APK analysis.
  • aegis scan <apk> — full static scan
  • aegis dynamic <apk> — sandbox execution
  • aegis report --format pdf|json
  • aegis device <id> --risk — device posture
  • Integrates with backend API via API key
  • Offline mode for air-gapped environments

Implementation Roadmap

Phase 1 — Weeks 1–4
Foundation: Agent + Ingestion
Deploy the Kotlin Android agent with work profile support via MDM enrollment. Implement device scanner modules (root detection, Play Integrity, patch level). Stand up the FastAPI ingestion gateway with mTLS and Kafka message bus. Define all telemetry schemas (Protobuf/Avro).
Kotlin Android WorkManager FastAPI Kafka Protobuf mTLS
Phase 2 — Weeks 5–8
Static Analysis Engine
Build the APK analysis pipeline: intake from S3, decompile with jadx, scan with custom rules. Implement secret detection (entropy + regex), permission abuse scoring, tracker SDK fingerprinting. Store results in PostgreSQL. Expose analysis API for CLI and dashboard.
apktool jadx MobSF PostgreSQL S3 Celery
Phase 3 — Weeks 9–13
Dynamic Sandbox + Behavior Fingerprinting
Deploy emulator cluster (Android x86 images in Docker). Integrate Frida server for hook injection. Capture runtime network, file, and crypto events. Build behavior signature generation pipeline. Implement behavior clustering to group similar malware families.
Frida Android Emulator Docker NGINX MinIO Redis
Phase 4 — Weeks 14–17
Risk Engine + SOC Dashboard
Implement the composite risk scoring model. Integrate VirusTotal and OTX threat intel feeds. Build the React SOC dashboard with real-time WebSocket updates. Add device fleet heatmap, alert timeline, and one-click MDM response actions.
Python React / Next.js WebSocket VirusTotal API Elasticsearch
Phase 5 — Weeks 18–20
Response Automation + Integrations
Wire up MDM integration (Intune / VMware Workspace ONE) for automated quarantine and policy push. Implement SIEM forwarding to Splunk and ELK. Add Jira auto-ticketing. Ship the CLI tool with full static + dynamic support and PDF report export.
Microsoft Intune API Splunk HEC Jira API Python CLI (Click) ReportLab PDF
Phase 6 — Weeks 21+
AI Analyst Layer + ML Hardening
Integrate LLM for automated threat report generation and SOC triage chat. Train ML classifier on labeled APK dataset for automated malware classification. Add anomaly detection on device behavioral baselines. Implement feedback loop from analyst verdicts.
Claude API scikit-learn MLflow LangChain RAG Pipeline

Differentiating Capabilities

01
Behavior Fingerprinting
Dynamic sandbox generates a cryptographic behavior signature per APK — a hash of its runtime API call graph. Enables detection of repackaged malware that changes code but retains behavior.
02
Local Risk Pre-scoring
The Kotlin agent calculates a preliminary risk score on-device and transmits only meaningful deltas — reducing bandwidth by ~80% versus raw telemetry dumps while preserving privacy.
03
App Reputation System
Internal "enterprise app store" intelligence layer. Every app accumulates a trust score with behavioral history across the entire fleet, enabling early anomaly detection for new deployments.
04
Permission Abuse Detector
Time-series analysis of permission usage patterns. Detects apps that suddenly request elevated permissions post-install or exercise permissions at abnormal frequencies.
05
Attack Path Visualization
SOC dashboard renders full kill-chain graphs — from app install to permission grant to API call to data egress — as interactive node graphs for rapid incident investigation.
06
LLM Threat Analyst
Analyst-facing AI that transforms complex static/dynamic analysis JSON into plain-English threat reports with severity context, business impact, and step-by-step remediation guidance.
