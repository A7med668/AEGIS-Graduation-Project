PH 02
Server Ingestion — All JSON arrives here first
🖥️ FastAPI Ingestion Server
Python 3.11 · Pydantic v2 · mTLS · PostgreSQL · MinIO/S3 — Central entry point for ALL device data

📥 Endpoints

  • POST /telemetry — DeviceReport JSON
  • POST /apps — AppSnapshot JSON
  • POST /logs — ImportantLogs JSON
  • POST /apk/upload — APK binary
  • GET /health · /metrics

🔐 Security

  • mTLS: device_id from cert CN
  • Rate limit: 10 req/min per device
  • APK: validate magic bytes only
  • Never log raw payloads

📤 Routing

  • Telemetry → PostgreSQL + Kafka
  • APK → MinIO/S3 + Kafka apk.analysis
  • Logs → Kafka logs.stream
  • Malicious flag → Kafka device.flagged
Malicious/flagged devices routed to analysis pipeline

PH 03
Analysis — Static Engine + AI CLI Agent (Parallel)
⚡ Run in Parallel — same phase, same trigger
🔬
Python Static Analyzer
apktool · jadx · Celery · Redis
  • APKUnpacker: decompile smali + Java/Kotlin
  • SecretDetector: API keys + entropy >4.5
  • PermissionAnalyzer: risk matrix (DANGEROUS/SIG/NORMAL)
  • TrackerDetector: Exodus Privacy DB
  • ObfuscationDetector: avg identifier <3 chars
  • → StaticReport JSON → risk.scores Kafka
🤖
AI CLI Agent (aegis scan)
Python · Click · Rich · Claude API
  • aegis scan <apk_path> — full local analysis
  • Receives same APK as Python engine
  • AI reviews decompiled code for logic threats
  • Color-coded Rich table output (risk 0–100)
  • aegis report: PDF/JSON/HTML output
  • → AI findings merged into StaticReport
Why parallel? Python static engine is fast and deterministic (regex, entropy, rules). AI CLI agent catches subtle logic threats and business-logic abuse that rules miss. Both run simultaneously on the same APK and their outputs are merged into one StaticReport.

PH 04
Log Streaming — AI Agent works on Important Logs only
🌊
Log Stream Processor
Kafka logs.stream · Python consumer
  • Consumes logs.stream Kafka topic (real-time)
  • Only processes: SECURITY · CRASH · ANOMALY tagged logs
  • All other logs discarded immediately — never stored
  • Batches important logs per device (5s window)
  • Dedup: skip same log fingerprint within 60s
  • → Forwards important batches to AI Agent
🧠
AI Log Analyst Agent
Claude API · claude-sonnet-4 · temp 0.2
  • Receives only pre-filtered important log batches
  • Detects: privilege escalation patterns
  • Detects: repeated auth failures / brute force
  • Detects: unusual process spawning
  • Detects: data exfiltration indicators
  • → LogThreatReport → risk.scores Kafka
PH 05
Risk Engine — Score · Alert · Respond
⚖ Risk Score Formula
RiskScore = (0.35 × StaticAnalysis) + (0.35 × DevicePosture) + (0.20 × LogThreats) + (0.10 × ThreatIntel)
DevicePosture: +40 rooted · +30 Play Integrity FAIL · +20 patch >90d · +10 bootloader unlocked
StaticAnalysis: +25 secrets found · +20 obfuscation · +15 trackers · +permission risk matrix · AI findings
LogThreats: AI LogThreatReport severity score (0–40 pts)
ThreatIntel: VirusTotal hash lookup · +50 if detections > 3 · Alert dedup: 4h window
🧮
Risk Engine
Python · PostgreSQL · VirusTotal
  • Aggregates all signals into RiskScore
  • VirusTotal hash lookup (async)
  • Alert dedup: 4h, +10pts threshold
  • Score ≥ 75 → trigger response.actions
