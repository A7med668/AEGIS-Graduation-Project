v4.4 Production Hardening quick notes
------------------------------------
Local demo:
1) Keep API_AUTH_ENABLED=false in .env
2) Run backend and frontend normally.

Production-style auth test:
1) Set API_AUTH_ENABLED=true
2) Set API_KEY=your-secret
3) Restart backend
4) In the UI go to Settings -> Production Auth and save the same key.

New endpoints:
- GET /api/queue/status
- GET /api/sandbox/policy

Production compose:
docker compose -f docker-compose.prod.yml --env-file .env.production up --build
