List of Changes

Version 1.9.1

- Handling of "scan path" messages from Music Player.
- Use git commit date and time for application information instead of system build time.
- Adapted to Android 16 (SDK 36).
- New Gradle and libraries.
- Android 4.4 (SDK 19) no longer supported, minimum is 5 (SDK 21).

Version 1.8

- Handling of ".nomedia" files repaired and made configurable.
- Music path can be selected via system file selector.

Version 1.7

- Ogg Opus file support added.
- Extract album pictures from Ogg Vorbis and Ogg Opus files.
- 1.7.1: SDK 33, new libraries, SQLite and TagLib updated.
- 1.7.2: Report number of album images found, debug functionalities added, landscape mode, reset paths moved to settings.

Version 1.6.1

- Correction: Precede movement name with movement number (arabic digits).
- Correction: After renaming album pictures, write correct (new) file name to database.
- Progress and result text corrected.

Version 1.6

- Vorbis (.ogg or .flac files) movement tags added according to the "MusicBrainz Picard" tagger.
- Preserve original file name of album art pictures when downscaling.
- Image files can be renamed automatically, e.g. from "folder.jpg" to "albumart.jpg".

Version 1.5

- Built for Android 10 (SDK 29).
- Base name for extracted album images is configurable ...
- ... and its default value is now "albumart" instead of "folder".
- Preference screen beautified.
- Warnings about failed album image extraction (mostly because SD card is only writable via SAF).
- Visual indication in case file access was denied.

Version 1.4

- Auto scan now finds added and removed album pictures.
- Multi CD album handling repaired.
- Extracted album pictures may be scaled, like existing ones.

Version 1.3

- SD card detection
- Database path is configurable
- Menu entry to restore default paths
- .nomedia file handling
- 1.3.1: Workaround for Android bug that causes a null path to be returned.

Version 1.2.4

- Smaller package
- Round app icon (for round displays)

Version 1.2.3

- Multi CD albums are processed correctly, i.e. with CD and track number.

Version 1.2.2

- Autoscan repaired, it used to fail for more than one added file.
- Extract conductor names from mp4 files.
- Taglib updated: 2019-03-27

Version 1.2

- Extract album covers from FLAC files.

Version 1.1

- Intelligent incremental scan added.
- Processes lists of changed files (sent from Classical Music Tagger) in background or foreground.

Version 1.0.2

- bug removed: numeric id3v2 genres are now handled.

Version 1.0.1

- bug removed: Year information was not extracted from mp3 files.
- bug removed: Track and CD numbers were not extracted from mp4 files.

Version 1.0

- First published version
